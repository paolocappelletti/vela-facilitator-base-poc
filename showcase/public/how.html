How it Works

A private X402 payment, step by step

Walk through the end-to-end flow of a private payment between buyer and seller on Base, executed inside the Horizen Vela TEE.

Start ↓
Step 1 · Preliminary

Buyer & seller register their keys

Both buyer and seller register their Base address to the Vela system by signing and sending a payload. In the example below they use the FacilitatorHelper — no ETH needed. Alternatively, they can use the Vela TypeScript library and call the Vela endpoint contract on Base directly. 

import { FacilitatorHelper } from "@horizen/x402-private-vela-fixed";

const helper = new FacilitatorHelper(facilitatorUrl, privateKeys);
const res = await helper.submit({
  requestType: REQUEST_TYPE_ASSOCIATEKEY,
  payload: buildPayload(publicKey),
  assetAmount: 0n,
  applicationId,
});

Step 2 · Deposit

The buyer deposits USDC into a private wallet

The buyer moves USDC from their public Base balance into a Vela-managed private account. Just like the previous step, this can be done by signing and sending a payload to the facilitator (no ETH needed) or with the Vela TypeScript library directly. 

import { FacilitatorHelper } from "@horizen/x402-private-vela-fixed";

const helper = new FacilitatorHelper(facilitatorUrl, privateKeys);
const res = await helper.submit({
  requestType: REQUEST_TYPE_PROCESS,
  payload: new Uint8Array(0), // deposit has an empty payload
  tokenAddress,
  assetAmount: amount,
  applicationId,
});
Continue → Step 3
Step 3 · x402 invoice

The seller protects an endpoint with x402

The seller responds to the buyer's HTTP request with a standard x402 402 Payment Required. The JSON shape is identical to the Coinbase x402 spec, with only two VaultPay-specific additions:

  • scheme: "private-vela-fixed"
  • an extra invoiceId field — a unique string the seller generates in any format they like 
// requirementsPayload comes from the seller's 402 response.
const requirementsPayload = {
  x402Version: 2,
  resource: { url: "http://resource-asked..." },
  accepts: [
    {
      scheme: "private-vela-fixed",
      network,
      asset: tokenAddress,
      amount: 100000,
      payTo: "0xsellerAddress",
      maxTimeoutSeconds: 60,
      extra: { invoiceId: "INVOICEID_XXXXX" },
    },
  ],
};

Step 4 · Payment payload

The buyer builds an encrypted paymentPayload

The buyer uses the standard x402Client() from Coinbase, extended with the VaultPay Vela plugin, to build an encrypted paymentPayload that carries everything needed to execute the payment inside the TEE. The requirementsPayload passed in is exactly the one received from the seller. 

import { x402Client } from "@x402/core/client"; 
import {registerPrivateVelaFixedClient} from "@horizen/x402-private-vela-fixed";

const buyerX402 = new x402Client();
await registerPrivateVelaFixedClient(buyerX402, { velaRpcUrl, privateKeys... });

const requirementsPayload: PaymentRequired = {
  x402Version: 2,
  resource: { url: "http://resource-asked..." },
  accepts: [requirementsPayload],
};
const paymentPayload = await buyerX402.createPaymentPayload(requirementsPayload);

Step 5 · Verify & settle

The seller settles via the Vela facilitator

The paymentPayload is delivered to the seller, who calls the Vela facilitator. He can use the standard HTTPFacilitatorClient from Coinbase — the Vela facilitator is 100% compatible with the standard API, no change required. If settlement succeeds, the payment has been completed inside the Vela TEE and the seller can serve the resource. 

import { HTTPFacilitatorClient } from "@x402/core/server";

const sellerFacilitator = new HTTPFacilitatorClient({ url: facilitatorUrl });

const verifyResult = await sellerFacilitator.verify(paymentPayload, requirements);
if (!verifyResult.isValid) {
  throw new Error(`/verify failed! ${verifyResult.invalidReason}`,
  );
}

const settleResult = await sellerFacilitator.settle(paymentPayload, requirements);
if (!settleResult.success) {
  throw new Error(`/settle failed: ${settleResult.errorReason}`,
  );
}

Step 6 · Withdraw

The seller withdraws funds when ready

At any time, the seller can withdraw funds from the private wallet back to their public Base balance — again either via the facilitator (no ETH needed) or by calling the Vela endpoint contract directly with the TypeScript library.
